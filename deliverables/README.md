## GitHub repo deliverable

Use your personal account on <https://github.com> to create a ***public*** GitHub repo to use for your project.

The GitHub repository used for the project must be named appropriately. For example, names that describe your project are appropriate, whereas generic names such as `ga-project` or `unit-project` are not.

## Deployed app deliverable

You'll need to deploy your project on the internet so that others are able to use it! Follow the [tktk Deployment guide](https://pages.git.generalassemb.ly/modular-curriculum-all-courses/universal-resources/deployment/tktk/) to deploy your project.

## Submitting your deliverables

Follow the guidelines below to submit your deliverables for this project.

### Submitting your planning materials

Submit the URL for your ***publicly accessible*** planning materials in the following format:

```plaintext
Project planning URL: <url>
```

Replacing `<url>` (including the `<` and `>`) with the URL for your project planning materials.

### Submitting your GitHub repo URL

Submit the URL for your ***public*** project GitHub repo in the following format:

```plaintext
GitHub repo URL: <url>
```

Replacing `<url>` (including the `<` and `>`) with the URL for your GitHub repo.

### Submitting the URL for your deployed app

Deploy your project by following the guide above, then submit the URL for your deployed app in the following format:

```plaintext
Deployed project URL: <url>
```

Replacing `<url>` (including the `<` and `>`) with the URL for your deployed app.
